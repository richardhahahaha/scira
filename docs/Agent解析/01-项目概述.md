# Scira 项目概述与 Extreme Search 功能

## 项目简介

**Scira** 是一个由 AI 驱动的极简搜索引擎，帮助用户在互联网上查找信息。它是研究 AI Agent 演化的绝佳案例，特别是其 **Extreme Search** 功能经历了从硬编码工作流到完整 AI Agent 的转型。

- **原始项目**：https://github.com/zaidmukaddam/scira
- **技术栈**：Next.js + Vercel AI SDK + 多种 AI 模型
- **核心特性**：AI 搜索、学术研究、代码执行、实时数据获取

## Extreme Search 功能定位

**Extreme Search** 是 Scira 的高级搜索功能，用于处理复杂的研究任务。与普通搜索不同，它能够：

- 📋 **自动规划研究步骤** - 将复杂问题分解为多个子任务
- 🔍 **多源信息收集** - 从网页、学术论文、社交媒体等多个渠道获取信息
- 🧪 **代码执行与分析** - 运行 Python 代码进行数据分析和可视化
- 📊 **实时数据可视化** - 生成图表展示研究结果
- 🔄 **自适应搜索策略** - 根据中间结果调整后续查询

## 为什么选择 Scira 作为教学案例？

### 1. **完整的演化路径**
Scira 的 Extreme Search 经历了清晰的演化过程：
```
硬编码 Workflow (v1) → 半自动化 (v2) → 完整 AI Agent (v3-当前)
```

### 2. **真实的生产级代码**
- 不是玩具项目，而是实际部署的应用
- 代码质量高，架构清晰
- 包含完整的错误处理和边界情况

### 3. **多维度的技术决策**
项目涉及：
- Agent 架构设计
- 工具集成（Tools）
- 流式输出（Streaming）
- 成本控制
- 用户体验优化

### 4. **可对比的实现方式**
通过 Git 历史可以看到：
- 旧版本的硬编码实现
- 新版本的 Agent 实现
- 两者的优缺点对比

## Extreme Search 的核心价值

### 用户视角
```
用户输入: "分析 2025 年 AI 芯片市场趋势"

传统搜索:
→ 返回 10 个搜索结果链接
→ 用户需要逐个点击阅读
→ 自己整理和分析信息

Extreme Search:
→ 自动分解为多个研究角度（市场份额、技术趋势、主要玩家等）
→ 从多个来源搜索信息（新闻、研究报告、社交媒体讨论）
→ 执行代码分析数据
→ 生成可视化图表
→ 提供综合分析报告
```

### 技术视角

**核心能力矩阵：**

| 能力 | 实现方式 | 难度 |
|------|---------|------|
| 任务规划 | LLM + Structured Output | ⭐⭐⭐ |
| 工具调用 | Vercel AI SDK Tools | ⭐⭐⭐⭐ |
| 自主决策 | Agent with Stop Conditions | ⭐⭐⭐⭐⭐ |
| 实时反馈 | Stream Writer | ⭐⭐⭐ |
| 代码执行 | Daytona Sandbox | ⭐⭐⭐⭐ |
| 数据可视化 | ECharts + Python | ⭐⭐⭐ |

## 快速开始

如果你想立即看到效果，可以：

1. **克隆项目**
```bash
git clone https://github.com/zaidmukaddam/scira.git
cd scira
```

2. **安装依赖**
```bash
pnpm install
```

3. **配置环境变量**
```bash
cp .env.example .env.local
# 填入必要的 API Keys (至少需要 XAI_API_KEY, EXA_API_KEY, DAYTONA_API_KEY)
```

4. **运行开发服务器**
```bash
pnpm dev
```

5. **测试 Extreme Search**
- 访问 http://localhost:3000
- 选择 "Extreme" 搜索模式
- 输入复杂查询如："分析 React 19 的新特性和性能改进"
